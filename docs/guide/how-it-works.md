# 工作原理

AgentChat 由三部分组成：

- **CLI**：在你的机器上运行 AI 编程代理
- **Hub**：保存状态、提供 API、协调消息同步
- **Web / PWA**：提供远程控制界面

## 架构总览

```text
┌──────────────┐   Socket.IO   ┌──────────────┐   REST + SSE   ┌──────────────┐
│ AgentChat CLI│◄────────────►│ AgentChat Hub│◄──────────────►│  Web / PWA   │
│ + AI Agent   │              │ + SQLite     │                │  手机 / 浏览器 │
└──────────────┘              └──────────────┘                └──────────────┘
        │                             │
        │ 本地进程                    │ http://localhost:3217
        ▼                             ▼
   项目文件 / 命令行              可选 HTTPS / 隧道
```

## 各组件负责什么

### AgentChat CLI

CLI 负责：

- 启动 Claude Code、Codex、Cursor、Gemini、OpenCode
- 将会话注册到 Hub
- 上报消息、状态、权限请求
- 暴露 AgentChat 的 MCP / RPC 能力

### AgentChat Hub

Hub 是中心协调层，负责：

- HTTP API
- CLI Socket.IO 连接
- SSE 推送浏览器更新
- SQLite 持久化会话、消息、机器信息
- 转发权限审批结果
- 提供 Web 页面和配套后端能力

### Web / PWA

Web / PWA 是远程控制面板，负责：

- 查看当前和历史会话
- 发送消息
- 审批权限
- 查看文件、终端、机器信息
- 从在线 runner 机器远程创建会话

## 典型数据流

### 新建会话

```text
1. 用户在终端运行 agentchat，或在网页里通过 runner 新建会话
2. CLI 启动对应 AI Agent
3. CLI 通过 Socket.IO 连接 Hub
4. Hub 保存会话元数据
5. 浏览器通过 SSE 收到会话更新
6. 手机 / 浏览器里出现新会话
```

### 权限审批

```text
1. Agent 请求一个权限
2. CLI 把请求发给 Hub
3. Hub 持久化并推送 SSE / 通知
4. 用户在网页或 PWA 中打开该会话
5. 用户批准或拒绝
6. Hub 再把结果转发回 CLI
```

### 消息往返

```text
浏览器 / 手机           Hub                 CLI / Agent
     │                   │                       │
     │ 发送消息          │                       │
     ├──────────────────►│                       │
     │                   ├──────────────────────►│
     │                   │      Socket.IO        │
     │                   │                       ├─ Agent 执行
     │                   │◄──────────────────────┤
     │      SSE 更新     │                       │
     ◄───────────────────┤                       │
```

## 本地模式与远程模式

### 本地模式

适合你在终端里专注工作时：

- 原生 Agent CLI 体验
- 响应最快
- 键盘驱动效率最高

### 远程模式

适合你离开终端但想继续控制会话时：

- 手机 / 浏览器可继续查看与发消息
- 远程审批权限
- 会话持续运行在你的机器上

### attach 模式

`agentchat attach <sessionId>` 允许你：

- 在终端附着到一个已经存在的会话
- 查看实时消息
- 从终端继续发送消息
- 但不会直接结束底层会话

## 为什么需要 Runner

Runner 的作用是让“网页端远程创建会话”成为可能。

没有 Runner 时：

- 你仍可以手动在终端启动本地会话
- 但网页端无法远程拉起新会话

有 Runner 时：

- Web 会看到在线机器
- 你可以从网页里指定目录和 Agent 类型，直接新建会话
